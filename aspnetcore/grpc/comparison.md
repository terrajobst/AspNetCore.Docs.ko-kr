### <a name="limited-browser-support"></a><span data-ttu-id="c6c24-183">제한된 브라우저 지원</span><span class="sxs-lookup"><span data-stu-id="c6c24-183">Limited browser support</span></span>

<span data-ttu-id="c6c24-184">현재 브라우저에서 gRPC 서비스를 직접 호출하는 것은 불가능합니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-184">It's impossible to directly call a gRPC service from a browser today.</span></span> <span data-ttu-id="c6c24-185">gRPC는 HTTP/2 기능을 많이 사용하며, 브라우저에서 gRPC 클라이언트를 지원하기 위해 웹 요청에 필요한 제어 수준을 제공하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-185">gRPC heavily uses HTTP/2 features and no browser provides the level of control required over web requests to support a gRPC client.</span></span> <span data-ttu-id="c6c24-186">예를 들어, 브라우저는 호출자가 HTTP/2를 사용하도록 요구하거나 기본 HTTP/2 프레임에 대한 액세스를 제공하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-186">For example, browsers do not allow a caller to require that HTTP/2 be used, or provide access to underlying HTTP/2 frames.</span></span>

<span data-ttu-id="c6c24-187">[gRPC-웹](https://grpc.io/docs/tutorials/basic/web.html)은 브라우저에서 제한적 gRPC 지원을 제공하는 gRPC 팀의 추가 기술입니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-187">[gRPC-Web](https://grpc.io/docs/tutorials/basic/web.html) is an additional technology from the gRPC team that provides limited gRPC support in the browser.</span></span> <span data-ttu-id="c6c24-188">gRPC-웹은 모든 최신 브라우저를 지원하는 JavaScript 클라이언트와 서버 상의 gRPC-웹 프록시의 두 부분으로 구성됩니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-188">gRPC-Web consists of two parts: a JavaScript client that supports all modern browsers, and a gRPC-Web proxy on the server.</span></span> <span data-ttu-id="c6c24-189">gRPC-웹 클라이언트는 프록시를 호출하고 프록시는 gRPC 요청을 gRPC 서버에 전달합니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-189">The gRPC-Web client calls the proxy and the proxy will forward on the gRPC requests to the gRPC server.</span></span>

<span data-ttu-id="c6c24-190">gRPC-웹에서 모든 gRPC 기능을 지원하지는 않습니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-190">Not all of gRPC's features are supported by gRPC-Web.</span></span> <span data-ttu-id="c6c24-191">클라이언트 및 양방향 스트리밍이 지원되지 않으며 서버 스트리밍이 제한적으로 지원됩니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-191">Client and bi-directional streaming isn't supported, and there is limited support for server streaming.</span></span>

### <a name="not-human-readable"></a><span data-ttu-id="c6c24-192">사람이 읽을 수 없음</span><span class="sxs-lookup"><span data-stu-id="c6c24-192">Not human readable</span></span>

<span data-ttu-id="c6c24-193">HTTP API 요청은 텍스트로 전송되며, 사람이 읽고 만들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-193">HTTP API requests are sent as text and can be read and created by humans.</span></span>

<span data-ttu-id="c6c24-194">gRPC 메시지는 기본적으로 Protobuf로 인코딩됩니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-194">gRPC messages are encoded with Protobuf by default.</span></span> <span data-ttu-id="c6c24-195">Protobuf는 송신 및 수신에 효율적이지만, 이진 형식은 사람이 읽을 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-195">While Protobuf is efficient to send and receive, its binary format isn't human readable.</span></span> <span data-ttu-id="c6c24-196">Protobuf를 사용하려면 올바른 역직렬화를 위해 *.proto* 파일에 지정된 메시지의 인터페이스 설명이 필요합니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-196">Protobuf requires the message's interface description specified in the *.proto* file to properly deserialize.</span></span> <span data-ttu-id="c6c24-197">네트워크에서 Protobuf 페이로드를 분석하고 요청을 직접 작성하려면 추가 도구가 필요합니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-197">Additional tooling is required to analyze Protobuf payloads on the wire and to compose requests by hand.</span></span>

<span data-ttu-id="c6c24-198">[서버 리플렉션](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md) 및 [gRPC 명령줄 도구](https://github.com/grpc/grpc/blob/master/doc/command_line_tool.md)와 같은 기능은 이진 Protobuf 메시지를 지원하기 위해 존재합니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-198">Features such as [server reflection](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md) and the [gRPC command line tool](https://github.com/grpc/grpc/blob/master/doc/command_line_tool.md) exist to assist with binary Protobuf messages.</span></span> <span data-ttu-id="c6c24-199">또한 Protobuf 메시지는 [JSON 변환](https://developers.google.com/protocol-buffers/docs/proto3#json)을 지원합니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-199">Also, Protobuf messages support [conversion to and from JSON](https://developers.google.com/protocol-buffers/docs/proto3#json).</span></span> <span data-ttu-id="c6c24-200">기본 제공 JSON 변환은 디버그할 때 Protobuf 메시지를 사람이 읽을 수 있는 형식으로 변환하는 효율적인 방법을 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-200">The built-in JSON conversion provides an efficient way to convert Protobuf messages to and from human readable form when debugging.</span></span>

<span data-ttu-id="c6c24-202">다음과 같은 시나리오에서는 gRPC보다 다른 프레임워크가 권장됩니다.</span><span class="sxs-lookup"><span data-stu-id="c6c24-202">Other frameworks are recommended over gRPC in the following scenarios:</span></span>
